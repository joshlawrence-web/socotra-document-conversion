# History — Strict Exact Matching

Append-only. Newest entry first.

## 2026-06-10 — Implemented

### What changed

- **`scripts/leg2_fill_mapping.py` — `match_token()`**: Removed Step 2 (case-insensitive entity/field lookup) and Step 3 (terminology synonym expansion). Only Step 1 (exact `EntityType` + `fieldName` match against schema index) is retained. Any non-exact result returns `[], "none", None`.

- **`scripts/leg2_fill_mapping.py` — `confidence_grade()`**: Simplified to: `exact` + `verified` → `high`; anything else → `none`. Removed `ci` from the high-confidence branch and the `terminology` → `medium` cap (D7). Old `low` return for non-verified exact matches is preserved.

- **`scripts/leg2_fill_mapping.py` — `suggest_loop_root()`**: Removed ci fallbacks (plural lookup, case-insensitive display_name lookup) and the terminology synonym expansion loop. Only an exact `iby_name` lookup where `cand.get("name") == loop_name` survives. Removed dead `alias` variable and unreachable `terminology` reasoning branch. Updated docstring: `match_step` is now `exact|none` only.

- **`scripts/leg2_fill_mapping.py` — enrichment loop (variables, loops, loop fields)**: All three loops now check `if <entry>.get("data_source"): continue` before attempting matching. An entry with `data_source` already populated is passed through unchanged — no matching, no pop, no overwrite. This is the hook for future explicit path provision at the docx/PDF stage.

### Motivation

User-directed hardening: the field catalog (`field-catalog.md`, generated by `list_paths`) is the authoritative menu of valid velocity paths. Users are expected to select paths explicitly from it. The pipeline should wire explicit selections, not infer them via heuristics.

### What was not changed

- `suggest_loop_field()` — coverage field matching via prefix decomposition; not name-heuristic-based, left intact.
- Leg 1 (`convert.py`) — still writes `data_source: ""` for all tokens. Explicit path provision at the docx/PDF stage is a future work item.
- `terminology.yaml` loader — infrastructure preserved; no longer consulted during matching.

### Verification

- `py_compile` passes on `leg2_fill_mapping.py` ✓
- No `ci` / `terminology` / `fuzzy` match steps reachable in updated code ✓
- Entries with explicit `data_source` will be skipped by all three enrichment loops ✓
